Lesson 1

Notes About the Web

  • HTML=hypertext markup language
  • HTML-->look of the website

Tags and Elements

  • Elements = the code that makes up the page
  • Tags described in this set of lessons:
    • p = paragraph tag
    • em = italics
    • br = line break
    • img = image (no closing tag)
    • a = anchor (for making links)

Compuers are Stoopid

  • Not literally; computers always do what you
    tell them, so a small error can mess up your
    code in a big way.

Lesson 2

HTML Structure and Browser Tools

  • "Tree-like" structure
  • Everything is a box
  • Use Sublime Text2 or something similar
  • Elements can be "inline" or "block", with the latter forming an invisible box around the text

HTML Structure

  • HTML text goes up and down the document in waves
  • Waves come from indentation
  • Indentation is a great way to maintain order to the page for debugging and editing

Lesson 3

CSS and styling

  • Avoid repetition--helps avoid errors (one reason to not do CSS in line with HTML code)
  • CSS--allows the adding of style to HTML code
  • Make sure files are linked in the same folder/organized for larger projects

Consistency and Repitition

  • CSS=Cascading Style Sheets
  • CSS enables you to create classes for elements to avoid repeating the same code over and over
  • Using the same styles for certain elements helps avoid errors, as well

Lesson 4

Programming and Python

  • Computers can be programmed to execute a specific sequence of steps
  • A computer program is a precise sequence of steps
  • Python=programming language run through an interpreter to execute instructions
  • Python "expressions" can take many forms e.g. print 1 + 5

Numbers and Strings in Python

  • Strings-->put in "">
  • Can index strings e.g. udacity[0] which would take the 'u' and print it
  • If you print "4" + "4" you would get 44
  • If you print 4 + 4 you'd get 8
